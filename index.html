How to Host Content on GitHub Pages

This guide explains step-by-step how beginners can host static content on GitHub Pages.

Step 1: Create a GitHub Account

  1. Visit https://github.com
  2. Click Sign up
  3. Complete registration and log in

Step 2: Create a New Repository

  1. Click + (top right) â†’ New repository
  2. Repository name: 
    yourusername.github.io
  3. Select Public
  4. Check Add a README file
  5. Click Create repository

Step 3: Add Your Website Files

Your website must contain an index.html file.

Option A: Upload Files

  1. Open your repository
  2. Click Add file â†’ Upload files
  3. Upload:
    • index.html
    • style.css (optional)
    • images (optional)
  4. Click Commit changes

Option B: Create index.html Manually

<html>
  <head>
    <title>My Website</title>
  </head>
  <body>
    <h1>Hello World</h1>
    <p>My first GitHub Pages site</p>
  </body>
</html>

Step 4: Enable GitHub Pages

  1. Go to Settings
  2. Click Pages
  3. Under Source:
    • Branch: main
    • Folder: /root
  4. Click Save

Step 5: Access Your Website

Your website URL:

https://yourusername.github.io/

Wait 1â€“2 minutes for deployment.

Step 6: Update Your Website

  1. Edit files in the repository
  2. Commit changes
  3. Refresh your website

Important Notes

  • Repository must be Public
  • index.html is mandatory
  • Only static content is supported (HTML, CSS, JS)
  • Backend is not supported
